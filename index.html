permutation

Posted on 2017-09-18

Time complexity is NP, so recursion should be used..

At first, I want to have a variavle named pos to record the postion of the added number, however, in this way, I cannot trace numbers backward. Everytime, I should iterate from 0 instead of the numbers after the position. But this will add numbers repeatedly, so an array used[n] is used to record whether the number has been added to the list.

In Permutation II, there are duplicate numbers so there might exist duplicate set if we use the code above. To avoid duplicate, I just used the function contains() in arraylist. However, TLE occurs again…

Then I think I can judge if the list is empty, if it is empty, duplicate numbers could not be added, but if it is not empty, the numbers could be added. However, this is not the case… duplicate 211 will occur in the given testcase.

The coorect judgement should be if used[i-1] is true, if the first duplicate element is not used but the second duplicate number is used, which means it will change the sequence of duplicate numbers added to the list and will lead to a duplciate sequence. 

class Solution {
    public List<List<Integer>> permuteUnique(int[] nums) {
        List<Integer> ls = new ArrayList<>();
        List<List<Integer>> res = new ArrayList<>();
        boolean[] used = new boolean[nums.length];
        Arrays.sort(nums);
        if(nums.length == 0)
            return res;
        permutation(nums,used,res,ls);
        return res;

    }
    public List<List<Integer>> permutation(int[] nums, boolean[] used, List<List<Integer>> res, List<Integer> ls)
    {
        if(ls.size() == nums.length)
        {
            res.add(new ArrayList<Integer>(ls));
            return res;
        }

        for(int i = 0; i < nums.length; i++)
        {
            if(!used[i])
            {
                if(i > 0 && nums[i] == nums[i-1] && !used[i-1])
                    continue;

                used[i] = true;
                ls.add(nums[i]);
                permutation(nums, used, res, ls);
                ls.remove(ls.size()-1);
                used[i] = false;

            }
        }
        return res;

    }
}

Trapping Rain Water

首先是我自己写的，不用动规的。先记录差值，寻找极大值点。再从极大值点向右找到第一个比该点大的点，容量就是极大值点减去中间的值。如果没找到比极大值点大的点，从最右往左找最大的点作为容器的高度。实现过程有很多复杂的小细节，太麻烦了！

大神解法：

  1. 两次遍历，一次存左边最大，一次存右边最大，然后比较左右中较小的作为容器的高度，计算存水量。
  1. 一次遍历即可，从两边往中间遍历。如果左值小于右值，右值不变，左值右移，如果最左点往右找到比最左值小的值，存水；否则，将比最左值小的值作为最左值。再次比较左右值大小。如果左值大于右值，右值左移，如果找到比最右值小的值，存水；否则，将比最右值大的值作为最右值。再次比较左右值大小。直到左右遇到。

leetcode-32

Posted on 2017-09-04

I spent very long time in this problem and trapeed in a wrong idea for hours. At last, I read other’s solution and learnt that the code is very concise. I was not sure how to process the situation where the left parentheses are more than right ones. If I solved this problem by splitting the string by left parenthesis, another mistake will apear when the right parenthesis is in front of left one. Finally, I wrote very long code without balancing these two problems.

The rightful dealing with left parenthesis more than right ones should not be splitting by left column, but only mark the position of the first left parenthesis after poping. There are three situations:

  1. ‘(‘ : just push into the stack
  2. ‘)’ : If the stack is empty, change the start position to the one right to the current position. Otherwise, pop, if the stack is not empty yet, longest string should be current position minus the position at top of the stack. Otherwise, longest string is current postion minus start position plus 1.
public class longesrValidParentheses {
    public static int longest(String s)
    {
        Stack<Integer> st = new Stack<>();
        int start = 0;
        int max = 0;
        for(int i = 0; i < s.length(); i++)
        {
            if(s.charAt(i) == '(')
                st.push(i);
            else{
                if(st.isEmpty())
                {
                    start = i + 1;
                }
                else {
                    st.pop();
                    if(!st.isEmpty())
                    {
                        max = Math.max(i-st.peek(),max);
                    }
                    else 
                    {
                        max = Math.max(i-start+1, max);
                    }
                }
            }

        }
        return max;
    }
    }

Hadoop笔记(二)：Hadoop的安装
安装Hadoop之前搜了很多关于Hadoop是什么的资料，大部分的资料都将Hadoop讲得神乎其神，然而看了这些文字之后仍然不懂Hadoop到底能干什么样的事情。于是我想要从例子出发，理解Hadoop到底是怎么样工作的。本节内容讲的是如何在Mac上安装Hadoop并且配置Eclipse，以及运行MapReduce的“hello world”程序–WordCount.
1.安装Hadoop：
安装的教程在网上都能找到,我参考的文章是：http://blog.csdn.net/u012336567/article/details/50778989。
安装Hadoop之前需要安装ssh，因为安装Hadoop需要远程登录。Mac OS X在“系统偏好设置”“共享”中的“远程登录”勾选ssh就可以了。然后运行下面命令行确认是否需要密码登录localhost：
$ ssh localhost
设置ssh免密码登录：
$ ssh-keygen -t dsa -P ‘’ -f ~/.ssh/id_dsa
第一行：ssh-keygen代表生成密钥，-t指定生成的密钥类型，dsa代表dsa密钥认证；-p提供密语，-f指生成的密钥文件
第二行：将公钥加入到用于认证的公钥文件中
$ cat ~/.ssh/id_dsa.pub >> ~/.ssh/authorized_keys
接下来就可以安装Hadoop了。主要的命令行是:
$ brew install hadoop(安装Hadoop)在Mac上不需要自己配置Java_Home因为brew在安装时已经配置好。
安装之后还要修改配置文件。配置过程参见上面的链接。
2.运行Hadoop：
跳转目录/usr/local/Cellar/hadoop/2.8.0/bin下：
$ hadoop namenode -format
再进入/usr/local/Cellar/hadoop/2.8.0/sbin,执行：
$ ./start-all.sh
访问一下网址可以查看Hadoop是否成功运行：
Resource Manager: http://localhost:50070
JobTracker: http://localhost:8088
Specific Node Information: http://localhost:8042
3.配置Eclipse:
下载puglins，在finder中右击eclipse图标，选择“显示包内容”，进入文件夹，将下载的压缩包解压，之后将其中的2.6.0版本的jar包拷贝到plugins文件夹中。此时，打开eclpse的“偏好设置”，发现里面有了Hadoop Map/Reduce选项，点开后，将/usr/local/Cellar/hadoop/2.8.0/libexec拷贝到Hadoop installation directory中。建立Hadoop项目就可以了。
4.运行WordCount程序:
WordCount代码网上可以找到，我现在只试着运行代码看结果，之后再去理解代码。运行的步骤如下：
在新建的Hadoop项目中建立class，将代码复制到class中，然后导出jar包(也可以直接运行程序，但是要输入参数，本文只讲导出jar包在命令行执行的过程)。然后将jar包放到主目录下，即/Users/**/,在后面的命令行中都要带着路径。执行一下命令行：
$ hadoop fs -mkdir /user/input
$ hadoop fs -put /usr/local/Cellar/hadoop/2.8.0/README.txt /input(将README.txt拷贝到input文件夹中，一定要添加路径)
$ hadoop jar /Users/apple/WordCount.jar hadoop2.WordCount /user/input output（运行程序，一定要添加各种路径,hadoop2是我的package名称,WordCount是class的名称）
$ hadoop fs -text output/part-r-00000(程序运行结束后，查看运行结果，结果是每个词的数量)
