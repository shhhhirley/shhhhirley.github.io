##FileSystem

In computer science, a B-tree is a self-balancing tree data structure that maintains sorted data and allows searches, sequential access, insertions, and deletions in logarithmic time. The B-tree is a generalization of a binary search tree in that a node can have more than two children. Unlike self-balancing binary search trees, the B-tree is well suited for storage systems that read and write relatively large blocks of data, such as discs. It is commonly used in databases and file systems.

Trapping Rain Water

首先是我自己写的。先记录差值，寻找极大值点。再从极大值点向右找到第一个比该点大的点，容量就是极大值点减去中间的值。如果没找到比极大值点大的点，从最右往左找最大的点作为容器的高度。实现过程有很多复杂的小细节，太麻烦了！

大神解法：

1.两次遍历，一次存左边最大，一次存右边最大，然后比较左右中较小的作为容器的高度，计算存水量。

2.一次遍历即可，从两边往中间遍历。如果左值小于右值，右值不变，左值右移，如果最左点往右找到比最左值小的值，存水；否则，将比最左值小的值作为最左值。再次比较左右值大小。如果左值大于右值，右值左移，如果找到比最右值小的值，存水；否则，将比最右值大的值作为最右值。再次比较左右值大小。直到左右遇到。

CNN Models

Posted on 2018-01-22

CNN Architecture：
LeNet, ALEX, VGG, GOOGLENET, RESNET

LeNet: Conv filter were $5\times5$ stride 1. Sampling(pooling) layers were $2\times2$ stride 2

AlexNet: deeper and much wider version of LeNet and won by a large margin the difficult ImageNet Large Scale Visual Recognition Challenge in 2012.
CONV1, 96 $11\times11$ filters at stride 4, pad 0
MAX POOLING 1, $3\times3$ filters at stride 2
NORM 1, normalization layer(not batchnorm)
CONV2, 256 $5\times5$ filters at stride 1, pad 2
MAX POOLING 2, $3\times3$ filters at stride 2
NORM2, normalization layer
CONV3, 384 $3\times3$ filters at stride 1, pad1
CONV4, 384 $3\times3$ filters at srtide 1, pad1
CONV5, 256 $3\times3$ filters at stride 1, pad1
MAX POOLING $3\times3$ filters at stride 1, pad1
FC6, 4096 neurons
FC7, 4096 neurons
FC8, 1000 neurons
Total number of parameters:
filter size number of filters $\times$ depth of images
(pooling layer has no parameters )

AlexNet Other properties:
-first use of ReLU
-used norm layers(not used anymore)
-heavy data augmentation(flipping, jittering, cropping, useful)
-dropout 0.5
-batchsize 0.5
-sgd momentum: 0.9
-learning-rate 0.0., reduce by 10
-L2 weight decay 5e-4
-7 CNN ensemble: 18.2%->15.4%

Typical architecture of CNN is:
[(CONV -> ReLu)$\times$M -> POOL]$\times$N -> FC$\times$K -> SOFTMAX

——————————— Typical & Recent Advance————————-

GoogleNet: ILSVRC 2014 winner. 22 layers. Its main contribution was the development of an Inception Module that dramatically reduced the number of parameters in the network. No FC layers. $12\times$ less than AlexNet.
Naive Inception module: apply parallel filter operations on the input from previous layer. Get multiple receptive field sizes(according to number of filters used) for convolution($1\times1, 3\times3, 5\times5$), then concatenate all filter outputs together depth-wise. This is very expensive computation.
Solution: “bottleneck” layers that use $1\times1$ convolutions to reduce feature depth. Input->$1\times1$conv to perform dimension reduction. Total number of product operation is reduced from 854M ops to 358M ops on $28\times28\times256$ input on the above different scale filters.

VGGNet: Smaller filters, deeper networks. Deeper networks: 8 layers(AlexNet) to 16-19 layers(VGG16Net and VGG19Net). Use ensembles for best results. Smaller filters: Only $3\times3$ CONV stride 1, pad 1, and $2\times2$ MAX POOL stride 2. Recpetive field(total spatial area of the input) of $7\times7$ stride 1 conv layer is the same as 3 $3\times3$ conv layers, but more layers, thus more non-linearities, and fewer parameters.

Is CNN really the deeper, the better? No. 56-layer model performs worse on both training and test error, so the deeper model performs worse, but it’s not caused by overfitting. Hypothesis: the problem is an optimization problem, deeper models are harder to optimize.
Solution: Use network layers to fit a residual mapping instead of directly trying to fit a desired underlying mapping, and this leads to the idea of ResNet.

ResNet: ResNet is a very deep networks using residual connections. It is 152-layer model for ImageNet. Residual Network was the winner of ILSVRC’15. ResNet are currently state of the art CNN models and the default choice for using ConvNets in practice.
Full ResNet Architecture:

  1. Stack residual blocks
  2. Every residual block has two $3\times3$ conv layers.
  3. Periodically, double number of filters and downsample spatially using stride2.
  4. Additional conv layer at the beginning.
  5. No FC layers at the end(only FC1000 to output classes).

Reference: Stanford CS231n
