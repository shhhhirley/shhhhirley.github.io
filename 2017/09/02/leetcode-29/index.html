Request：

Divide two integers without using multiplication, divisor and mod operator. If it is overflow, return MAX_INT.

An idea came to my mind that I should use bit operation, but I am not sure how to use it, and I haven’t used bit operation in programming before. Besides, I am not familiar with solving overflow problems. I should have more practices on this kind of problems.

After reading blogs in C++, I learnt the idea that for a dividend, it must be between the interval of $$$2^k$$$ and $$$2^{(k+1)}$$$. This interval might be a very large interval with the increase of k that there are many other feasible solutions between them. So we need to add a loop to include these solutions after we determine what k is.

The function given by leetcode has args of int type, so the args themselves cannot overflow, only after the operation, overflow might occur. Thus, there are only two cases we should consider of, that is, when dividend is MIN_VALUE and divisor is -1, since the absolute value of MAX_VALUE is 1 less than that of MIN_VALUE.
